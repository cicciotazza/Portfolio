Projects

Pokedéx

Pokédex is an application built with Javascript, HTML, CSS and Bootstrap. Visitors can use the search bar to look for a specific Pokémon inside a collection loaded from an external API. More details about the single Pokemon are available by cliking on the name and displayed in a modal pop-up.

myFlix API

The serverside of myFlix app where it's possibile to make CRUD operations on the database via Postman. Movies, directors and genres can be checked, also register a new user, loggin in, add a movie to a list of favourites etc Developed using Node.js, Express and Mongoose. It's built using the MERN stack.

myFlix React

The client-side of myFlix API which fetches data from the server hosted on Heroku. It's the graphical unit interphace developed using React, React Bootstrap and React Redux and built using the MERN stack. Users can register themselves, logging in, checking the catalogue, update the profile etc.

Meet App

Serverless and progressive web application (PWA) built with React using a test-driven development (TDD) technique. The app uses the Google Calendar API to fetch upcoming events and AWS Lambda as the authorisation server. Available offline through a service worker. Data visualization using a Recharts library, a scatter graph and a pie chart.

Chat app

A mobile chat app built using React, React Native, and Gifted Chat to let users chat with each other. It's possible to pick a background color, read/send messages, share images and location. The app uses also Async Storage to allow users to view stored messages when offline and the Google service Firestore to store data, such as messages and images.

myFlix client - Angular

Another frontend of myFlix API created using Angular. This application allows users to register a profile, log in, consult a list of movies. Yet it offers the chance to see the genre, director description of each movie considered. Also, favoutires movies can be added to a list and deleted. The app has been built using Angular, Angular Material, Typescript and and Typedoc documentation has been added.
